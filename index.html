git的基本操作

1.创建本地仓库

创建.gitignore 配置文件用于设置提交时的忽略文件

git init 将文件夹变为git文件夹

git add . 将所有文件提交到暂存区

git commit -m '(提交描述)' 将文件提交到工作区

2.创建远程仓库

在github上New Repository

3.将本地仓库推送到远程仓库

git remote add origin url 关联远程仓库

git push origin master(推送的分支) 推送

4.远程拉取到本地

git pull origin master 拉取到当前分支

git fetch origin master:tmp 拉取到新的分支tmp

5.克隆远程仓库

git clone url 克隆

6.git分支操作

git branch 查看分支

git branch dev 创建dev分支

git checkout dev 切换到dev分支

git diff master dev 比较两个分支

git merge dev 将dev分支合并到当前分支

7.git的几个主要分支

master分支 主分支

develop分支 开发分支

release分支 测试分支

feature分支 功能特性分支

8.区别fork与clone

fork是将别人的远程仓库完全复制为自己的远程仓库

clone将远程仓库完全复制为本地仓库

9.区别git的pull和fetch

pull拉取的远程仓库到本地会与当前分支进行自动合并

fetch拉取的远程仓库会拉取到一个新的分支上，后续手动执行合并操作

模块化

对模块化的理解

①如果一个项目开发的足够大的时候，所有的js代码定义在一个js文件中的话，会使得复杂度提升，后期维护的难度也会增大

②如果将一个大的js文件根据一定的规范分成几个小的文件的话会便于管理，可以提高复用性

③模块化在项目中十分的重要，一个复杂的项目肯定有很多相似的功能模块，重新编写费事费力，引入别人的前提就需要引出模块化规范的使用。

④常用的JavaScript模块化规范有四种：Commonjs、AMD、CMD、ES6

Commonjs

Commonjs是服务器模块化的规范
采用了这个规范。但目前也可用于浏览器端
需要使用Browserify/webpack进行提前编译打包

Commomjs暴露的方式：

    1.module.exports = value;

    2.exports.xxx = value

Commomjs引入的方式：

    let 变量名 = require(文件路径)

暴露的本质是exports对象

ES6

ES6模块化规范专门针对于浏览器端，但目前浏览器支持不是很好(只有谷歌支持)，也需要使用Browserify进行打包

ES6暴露的方法：

    1.export xxx(常规暴露,暴露的本质是对象，接收的时候只能以对象的解构赋值的方式来接收)

    2.export default(默认暴露，暴露任意数据类型，暴露什么数据类型就接收什么数据类型)

ES6引入的方法：

    import {}/变量名 from 文件地址

实现：

ES6→ES5：bable 源文件路径 -d 编译后文件路径
编译打包js：browserify源文件路径 -o 编译后文件路径

webpack核心概念

Entry：

入口，webpack进行打包的起始点

Output：

出口，webpack编译打包生成的bundle文件

Loader：

模块加载器，将非js模块包装成webpack能理解的js模块

Plugin：

插件，在webpack构建流程中的特定时机插入具有特定功能的代码

Module：

模块，在webpack中一切皆模块，默认只识别js文件，如果是其他类型的文件要用对应的loader转化为js模块

常用数组操作方法

concat()：arr=arr1.concat(arr2)，用于连接两个数组

arr.slice(start,end)：返回一个从start到end(不包括end)的新数组

arr.splice(start,length,num)：从start开始，将长度为length的元素替换为num(当length为0时实现替换操作)

arr.sort()：数组排序(升序，[1,2,11,21]→[1,11,2,21])

arr.reverse()：颠倒数组中元素顺序

arr.filter()：数组过滤，返回结果为true的数据数组

arr.map()：便利数组，根据return生成新数组

arr.find()：找到数组中符合当前搜索规则的第一个元素

arr.from()：将伪数组转换为真数组

对cookie理解

分类：

会话cookie和持久化cookie

创建：

由服务器端创建，res.cookie(key,value,{name:tom})

保存：

由浏览器端保存，浏览器接收到新的cookie会自动保存在内存或者文件中

使用：

浏览器发送请求时，自动携带对应的cookie，服务器端通过req读取(req.cookies.key)
